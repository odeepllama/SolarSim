Usage & Help

Quick reference for building and managing profiles. Press Esc to close.

Monitor / Interact

  • Status Tiles: Click any tile for presets or custom values.

    Examples: SimTime (Sunrise/Noon or HHMM), Speed (−300x to 300x or Hold), Intensity (0.5–3.0). Double-click to toggle Program/Rotation/Dual Sun.

  • Solar Arc: Visual sun path showing current time and day/night shading.

    Click anywhere on the arc to jump the device to that time. Current device time is shown with an indicator line. Double-click the sun icon to toggle between Rotation and Program modes.

  • Command Console: Type commands with autocomplete support.

    Press Tab to cycle through autocomplete suggestions. Use Up/Down arrows to navigate your last 10 commands. Press Enter to send a command. Commands are case-sensitive and control device behavior directly.

  • Show All Commands: Displays full list of available commands in the log.

    Temporarily disables scroll lock to show output. Common commands: status (device state), start (begin program), stop (halt program), set autoload on/off (control auto-startup behavior).

  • Scroll Lock: Toggle auto-scroll in the command log.

    When ON (red), the log stays at your current scroll position. When OFF, it auto-scrolls to show the newest messages. Use this to review older output without being pushed down by new messages.

  • Device Connection: Web Serial API connects directly to your device via USB.

    Requires a compatible browser (Chrome, Edge, Opera). Click the "Connect" button and select your device from the list. Ensure proper USB driver installation if connection fails. The status shows "Connected" when active and "Disconnected" otherwise.

Profile Management

  • Recent Profiles: Quick access to previously used profiles.

    Auto-saved when you Download, Upload, Import, or select a profile. Click a grey button to load it into the form, click ★ to pin at top (pinned profiles survive browser data clear), or click ✕ to delete. Pinned profiles are useful for frequently-used configurations.

  • ⚡ Load Current: Quickly load the profile currently running on the device.

    Enabled when the device reports its loaded profile name. This is a convenience feature to fetch the active profile without browsing the device file list. The profile loads into the form so you can edit and re-upload if needed.

  • Load from Device: Browse and select profiles stored on the device.

    Selecting a profile immediately runs it on the device for this session only. To make a profile persistent across power cycles, click "Upload to Device" after loading. The device will auto-load the newest timestamped profile on next startup unless you disable auto-load.

  • Upload to Device: Send the current form settings to the device.

    Creates a new timestamped profile file on the device and immediately runs it. This becomes the "newest" profile for future auto-load on startup. Use this when you've made changes to a profile and want them to persist.

  • Download Profile: Save the current form as a JSON file to your computer.

    Creates a .json file with all current settings. Useful for backup, sharing, or manual editing. Import the file later to restore settings. Downloaded files are also added to Recent Profiles automatically.

  • Import Profile: Load a previously saved .json profile file.

    Updates the form only—does NOT change device behavior until you click "Upload to Device." Imported profiles are added to Recent Profiles for quick access. This is useful for testing settings before uploading to hardware.

  • Profile Diff Viewer: Visual highlighting shows when form differs from device.

    When you load a profile into the form (via Recent Profiles, Import, or editing settings), any fields that differ from what's currently on the device are highlighted with a yellow background and orange border. Small orange badges next to field labels show the device's current value for quick comparison.

    Diff Banner: When differences are detected, a banner appears showing the count of changed fields with action buttons:

    • Show First Difference: Scrolls to and highlights the first changed field
    • Sync to Device: Quick shortcut to upload your current form settings
    • Dismiss: Hides the banner temporarily (reappears if you make new changes)

    When highlighting appears: After loading profiles, importing files, editing form fields, or clicking recent profile buttons.

    When highlighting clears: After successfully uploading to device, loading from device, or when form values match device state.

    Tip: The diff viewer helps prevent accidentally overwriting device settings. Yellow highlighting means "this will change on the device if you upload."

  • Profile Comparison Modal: Preview differences before loading a profile.

    When you click a Recent Profile button, a comparison modal appears showing a side-by-side view of your current form settings versus the profile you're about to load. This lets you review what will change before applying.

    Comparison View: The modal displays settings organized into sections (Core Settings, Program Settings, Rotation & Imaging, Program Steps, etc.). Fields that differ between your current form and the selected profile are highlighted with a yellow background.

    Diff Summary: At the bottom of the modal, you'll see a count of how many fields differ (e.g., "5 fields differ"). If profiles are identical, it shows "Profiles are identical" in green.

    Actions:

    • Apply Profile: Load the selected profile into your form, replacing current settings
    • Cancel: Close the modal without making changes
    • X (close): Same as Cancel - dismisses the comparison

    Column Headers:

    • Current Form: Your existing form values
    • Profile Name: The values from the profile you're comparing

    Tip: Use this to understand exactly what will change when switching between profiles. Particularly useful when working with similar experimental setups that differ in just a few parameters.

  • Save/Apply/Clear Default: Manage your baseline configuration.

    Save Default: Store current form as your preferred baseline. Apply Default: Restore the form to your saved baseline. Clear Default: Remove the baseline (resets to empty form). Defaults are stored locally in your browser.

  • Reset Dialogs: Clear "Don't ask again" preferences.

    If you previously checked "Don't ask again" on Load or Upload dialogs, use these buttons to re-enable confirmation dialogs. Each type (Load, Upload) has its own reset button.

Settings & Configuration

  • Core Settings: Essential parameters for sun simulation.

    Time: Current simulated time (HHMM, 24-hour). Solar: Latitude, longitude, and date to calculate sun path. Intensity: Light multiplier (0.5–3.0). Rotation: Turntable angle and speed.

  • Program Settings: Enable step-based sequences.

    PROGRAM_ENABLED: Toggle step sequences on/off. PROGRAM_REPEATS: -1 (infinite loop), 1 (run once), or N (run N times). Program Auto-Start: Run automatically on device startup, or wait for manual command.

  • Undo/Redo: Revert or restore recent changes.

    Use Ctrl-Z (Windows/Linux) or Cmd-Z (Mac) to undo changes. Use Ctrl-Y or Cmd-Y to redo. Works for all form fields and program step edits. Maintains history of last 10 actions across settings changes, step additions, deletions, and reordering.

  • Validation & Safety: Rules to prevent device damage.

    Errors are shown inline in red; exports are disabled until resolved. Dual Sun with high intensity is blocked to prevent overheating. Intensity outside 0.5–3.0 is invalid. Realistic ranges for research: intensity 0.8–1.2, speed 1x–4x.

Program Steps

Define a sequence of timed sun position and light intensity changes.

Drag the ⠿ handle to reorder rows. Each step controls:

  • sim_time_hhmm: Target time (HHMM, 24h)
  • transition: RUN (time advances) or JUMP (instant, no duration)
  • speed: Time scale (negative = reverse, 0 = hold). Leave empty to keep current.
  • intensity_scale: Light multiplier 0.0–3.0 for this step. Leave empty to use global setting.
  • dual_sun: Toggle mirrored sun for this segment
  • hold_minutes: Real-time pause duration
  • repeat: Run this step N additional times before moving to the next

Timeline Visualization

Visual preview of your program sequence with color-coded steps.

Color Legend: Blue ≤1x, Green ≤2x, Yellow ≤6x, Orange 7x+. Saturation: Pale = low intensity, vibrant = high intensity. Markers: ⚡ (JUMP), hatched (HOLD), ↺ (day wrap). Summary: Shows net simulated span + estimated real runtime. If timeline looks wrong, check that steps are in chronological order and transitions are valid.

Workflow

  1. Configure core settings and enable Program if using steps
  2. Add/edit steps; review timeline and validation
  3. Upload to Device (Web Serial) or Download to save locally
  4. Optionally Save as Default for quick restore

Profile Loading & Running Regimes

Understanding Form vs Device

Key Concept: The HTML form and physical device are separate systems:

  • Form State: What you see/edit in the web interface
  • Device State: What's actually running on the hardware
Profile Loading Methods
1. Recent Profiles / Import Profile
  • Updates: HTML form only (green highlighting)
  • Does NOT affect: Device behavior
  • Purpose: Load settings for editing before upload
  • Next Step: Click "Upload to Device" to send to hardware
2. Load from Device
  • Updates: Both form AND device
  • Process: Sends loadprofile command to device
  • Result: Device immediately runs the selected profile
  • Duration: Temporary - only until device power cycle/restart
  • Auto-load: Unchanged - will revert to newest timestamped profile on restart
3. Upload to Device
  • Sends: Current form settings to device
  • Creates: New profile file on device
  • Result: Device runs the uploaded profile
  • Side Effect: Re-enables auto-load for future startups
Auto-Run Settings
Auto-Load on Startup
  • When: Device powers on or resets
  • What: Automatically loads the newest timestamped profile_*.txt file
  • Logic: Lexicographic sort - 20251003_2104_* comes after 20251003_2016_*
  • Persistence: Always loads newest timestamp, regardless of "Load from Device" selections
  • Control: set autoload on/off command creates autoload.cfg file on device
  • Config File: autoload.cfg persists setting across power cycles (contains '1' or '0')
Program Auto-Start
  • Enabled: Loaded profile immediately begins running program steps
  • Disabled: Profile loads but waits for manual start command
  • Use Cases: OFF for development/testing, ON for unattended operation
Common Workflows
Scenario 1: Modify Existing Profile
  1. Click grey recent profile button → Loads into form
  2. Edit settings as needed
  3. Click "Upload to Device" → Sends modified profile to hardware
  4. Device runs the updated profile
Scenario 2: Run Specific Old Profile (Temporary)
  1. Click "Load from Device" → Shows available profiles
  2. Select desired profile → Device immediately loads and runs it
  3. Temporary only - profile runs until power cycle
  4. After restart → Device reverts to newest timestamped profile
Scenario 3: Make Old Profile Persistent
  1. Click "Load from Device" → Select old profile → Loads into form
  2. Click "Upload to Device" → Creates new timestamped file
  3. New timestamp makes it the "newest" for auto-load
  4. Future restarts will auto-load this profile
Scenario 4: Quick Profile Switch (Session Only)
  1. Use "Load from Device" to switch between stored profiles
  2. Each selection immediately changes device behavior
  3. No upload needed - profiles already exist on device
  4. Reset warning - power cycle reverts to newest timestamped profile
Troubleshooting
  • Form shows profile but device not responding: Profile only loaded in form - use "Upload to Device"
  • Wrong profile runs after restart: Auto-load uses newest timestamped profile, not last selected. Upload desired profile to make it newest.
  • Old profile stops running after power cycle: "Load from Device" is temporary. Upload the profile to make it persistent.
  • Profile loads but doesn't start: Program Auto-Start may be disabled - check device settings
  • Green highlighting disappears: Normal when device status updates - highlighting shows form state vs device state
  • Auto-load keeps loading wrong profile: Use "set autoload off" to disable (creates autoload.cfg), or upload correct profile with newer timestamp

Program Step Editing

Program Step Behavior:
  • Speed: Applies from step's start until next speed change
  • RUN transitions: Advance simulated time using active speed
  • Backward RUN: Negative speed runs to earlier time/day when next step is later
  • JUMP transitions: Instantaneous reposition, no simulated time added
  • RUN/JUMP to same or earlier time: Creates new day
  • HOLD (Speed=0): Adds real-time duration only
