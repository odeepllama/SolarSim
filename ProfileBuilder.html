Interface Controls

  • Device Label: Click the badge at the top center of the page (e.g., "Device 1") to customize the device name.

    When working with multiple devices across different browser tabs, you can give each tab a unique identifier. Click the badge to set a custom name like "Device 1", "Device 2", "Lab A", etc.

    The device label appears in both the page badge and the browser tab title, making it easy to distinguish between multiple devices. The label is saved per tab (not globally), so each browser tab can have its own device identity.

    Use Case: Open multiple tabs, each connected to a different physical device, and label them accordingly to avoid confusion.

  • Background Color: Click colored swatches (top right) to change the page background color.

    Useful for visually distinguishing between multiple device tabs—assign each device a different color (green, blue, or red) to quickly identify which tab you're working in. Your color preference is saved per browser tab.

  • Simple/Advanced Mode: Toggle between streamlined and full interface views.

    Simple Mode: Hides less commonly used sections for a cleaner interface. Ideal for routine operations or when you're already familiar with the system.

    Advanced Mode: Shows all available sections and settings. Use this when you need full control or are learning the system.

    Gear Icon (⚙): Click to customize which sections are visible in Simple mode. Your preferences are saved automatically.

    Tip: Start with Advanced mode to explore all features, then switch to Simple mode once you know which sections you use regularly.

Monitor / Interact

  • Status Tiles: Click any tile for presets or custom values.

    Examples: SimTime (Sunrise/Noon or HHMM), Speed (−300x to 300x or Hold), Intensity (0.0–1.0). Double-click to toggle Program/Rotation/Dual Sun.

  • Solar Arc: Visual sun path showing current time and day/night shading. Also models sun position, colour, and intensity.
  • Command Console: Type commands with autocomplete support.

    Press Tab to cycle through autocomplete suggestions. Use Up/Down arrows to navigate your last 10 commands. Press Enter to send a command. Commands are case-sensitive and control device behavior directly.

  • Show All Commands: Displays full list of available commands in the log.
  • Device Connection: Web Serial API connects directly to your device via USB.

    Requires a compatible browser (Chrome, Edge, Opera). Click the "Connect" button and select your device from the list. Ensure proper USB driver installation if connection fails. The status shows "Connected" when active and "Disconnected" otherwise.

Profile Management

  • Recent Profiles: Quick access to previously used profiles.

    Auto-saved when you Download, Upload, Import, or select a profile. Click a grey button to load it into the form, click ★ to pin at top (pinned profiles survive browser data clear), or click ✕ to delete. Pinned profiles are useful for frequently-used configurations.

  • Load from Device: Browse and select profiles stored on the device.

    Selecting a profile immediately runs it on the device for this session only. To make a profile persistent across power cycles, click "Upload to Device" after loading. The device will auto-load the newest timestamped profile on next startup unless you disable auto-load.

  • Upload to Device: Send the current form settings to the device.

    Creates a new timestamped profile file on the device and immediately runs it. This becomes the "newest" profile for future auto-load on startup. Use this when you've made changes to a profile and want them to persist.

  • Download Profile: Save the current form as a .txt file to your computer.

    Creates a .txt file with all current settings in a format compatible with the device. Useful for backup, sharing, or manual editing. Import the file later to restore settings. Downloaded files are also added to Recent Profiles automatically.

  • Import Profile: Load a previously saved .txt profile file.

    Updates the form only—does NOT change device behavior until you click "Upload to Device." Imported profiles are added to Recent Profiles for quick access. This is useful for testing settings before uploading to hardware.

  • Profile Diff Viewer: Visual highlighting shows when form differs from device.

    When you load a profile into the form (via Recent Profiles, Import, or editing settings), any fields that differ from what's currently on the device are highlighted with a yellow background and orange border. Small orange badges next to field labels show the device's current value for quick comparison.

    Diff Banner: When differences are detected, a banner appears showing the count of changed fields with action buttons:

    • Show First Difference: Scrolls to and highlights the first changed field
    • Sync to Device: Quick shortcut to upload your current form settings
    • Dismiss: Hides the banner temporarily (reappears if you make new changes)

    When highlighting appears: After loading profiles, importing files, editing form fields, or clicking recent profile buttons.

    When highlighting clears: After successfully uploading to device, loading from device, or when form values match device state.

    Tip: The diff viewer helps prevent accidentally overwriting device settings. Yellow highlighting means "this will change on the device if you upload."

  • Profile Comparison Modal: Preview differences before loading a profile.

    When you click a Recent Profile button, a comparison modal appears showing a side-by-side view of your current form settings versus the profile you're about to load. This lets you review what will change before applying.

    Comparison View: The modal displays settings organized into sections (Core Settings, Program Settings, Rotation & Imaging, Program Steps, etc.). Fields that differ between your current form and the selected profile are highlighted with a yellow background.

    Diff Summary: At the bottom of the modal, you'll see a count of how many fields differ (e.g., "5 fields differ"). If profiles are identical, it shows "Profiles are identical" in green.

    Actions:

    • Apply Profile: Load the selected profile into your form, replacing current settings
    • Cancel: Close the modal without making changes
    • X (close): Same as Cancel - dismisses the comparison

    Column Headers:

    • Current Form: Your existing form values
    • Profile Name: The values from the profile you're comparing

    Tip: Use this to understand exactly what will change when switching between profiles. Particularly useful when working with similar experimental setups that differ in just a few parameters.

  • Save/Apply/Clear Default: Manage your baseline configuration.

    Save Default: Store current form as your preferred baseline. Apply Default: Restore the form to your saved baseline. Clear Default: Remove the baseline (resets to empty form). Defaults are stored locally in your browser.

  • Reset Dialogs: Clear "Don't ask again" preferences.

    If you previously checked "Don't ask again" on Load or Upload dialogs, use these buttons to re-enable confirmation dialogs. Each type (Load, Upload) has its own reset button.

Settings & Configuration

  • Core Settings: Essential parameters for sun simulation.

    Start Time: Initial simulated time (HHMM, 24-hour format). Time Scale: Speed multiplier for time progression. Intensity Scale: Global light multiplier (0.0–1.0). Simulation Date & Latitude: Geographic parameters for sun path calculations. Solar Mode: BASIC (constant) or SCIENTIFIC (astronomical calculations). Sun Color Mode: Color temperature settings (BLUE, NATURAL, or CUSTOM RGB).

  • Program Settings: Enable step-based sequences.

    PROGRAM_ENABLED: Toggle step sequences on/off. PROGRAM_REPEATS: -1 (infinite loop), 1 (run once), or N (run N times). Program Auto-Start: Run automatically on device startup, or wait for manual command.

  • Undo/Redo: Revert or restore recent changes.

    Use Ctrl-Z (Windows/Linux) or Cmd-Z (Mac) to undo changes. Use Ctrl-Y or Cmd-Y to redo. Works for all form fields and program step edits. Maintains history of last 10 actions across settings changes, step additions, deletions, and reordering.

  • Validation & Safety: Rules to prevent device damage.

    Errors are shown inline in red; exports are disabled until resolved. Dual Sun with high intensity is blocked to prevent overheating. Intensity above 1.0 is not permitted. Recommended ranges for research: intensity 0.8–1.0, speed 1x–4x.

Program Steps

Define a sequence of timed sun position and light intensity changes.

Program steps allow you to automate sun simulation and device actions over time. Each row in the table represents a step with specific parameters:

  • sim_time_hhmm: Target simulation time (HHMM, 24-hour format)
  • transition: RUN (advance time) or JUMP (instant move, no duration)
  • speed: Time scale (negative = reverse, 0 = hold). Leave empty to keep current speed.
  • intensity_scale: Light multiplier (0.0–1.0) for this step. Leave empty to use global setting.
  • dual_sun: Enable mirrored sun for this segment
  • hold_minutes: Real-time pause duration (only applies if speed=0)
  • repeat: Repeat this step N additional times before moving to the next

Step and Day Navigation:

  • Use the Step navigation buttons to move between individual steps in the timeline.
  • Use the Day navigation buttons to jump between days when your program spans multiple simulated days.
  • Navigation controls are located above the timeline and allow quick review and editing of specific segments.
  • Dragging the ⠿ handle reorders steps; you can also drag day boundaries to adjust timing.

Validation ensures correct formats and prevents unsafe device actions. Errors are shown inline and must be resolved before exporting or uploading.

Timeline Visualization

Visual preview of your program sequence with color-coded steps.

The timeline provides a graphical overview of your program steps, showing how simulated time and device actions progress:

  • Color Legend: Blue ≤1x, Green ≤2x, Yellow ≤6x, Orange 7x+. Saturation indicates intensity (pale = low, vibrant = high).
  • Markers: ⚡ (JUMP transitions), hatched bars (HOLD segments), ↺ (day wrap/new day).
  • Step Navigation: Use navigation buttons to move the playhead between steps and days for focused editing and review.
  • Drag to Edit: Drag segment boundaries or entire segments to adjust timing and order when not running on device.
  • Summary: Timeline shows total simulated span and estimated real runtime for your program.
  • If the timeline looks incorrect, check that steps are in chronological order and transitions are valid.

Timeline navigation and editing tools help you quickly visualize and refine complex step sequences for accurate device simulation.

Workflow

  1. Configure core settings and enable Program if using steps
  2. Add/edit steps; review timeline and validation
  3. Upload to Device (Web Serial) or Download to save locally
  4. Optionally Save as Default for quick restore

Profile Loading & Running Regimes

Understanding Form vs Device

Key Concept: The HTML form and physical device are separate systems:

  • Form State: What you see/edit in the web interface
  • Device State: What's actually running on the hardware
Profile Loading Methods
1. Recent Profiles / Import Profile
  • Updates: HTML form only (green highlighting)
  • Does NOT affect: Device behavior
  • Purpose: Load settings for editing before upload
  • Next Step: Click "Upload to Device" to send to hardware
2. Load from Device
  • Updates: Both form AND device
  • Process: Sends loadprofile command to device
  • Result: Device immediately runs the selected profile
  • Duration: Temporary - only until device power cycle/restart
  • Auto-load: Unchanged - will revert to newest timestamped profile on restart
3. Upload to Device
  • Sends: Current form settings to device
  • Creates: New profile file on device
  • Result: Device runs the uploaded profile
  • Side Effect: Re-enables auto-load for future startups
Auto-Run Settings
Auto-Load on Startup
  • When: Device powers on or resets
  • What: Automatically loads the newest timestamped profile_*.txt file
  • Logic: Lexicographic sort - 20251003_2104_* comes after 20251003_2016_*
  • Persistence: Always loads newest timestamp, regardless of "Load from Device" selections
  • Control: set autoload on/off command creates autoload.cfg file on device
  • Config File: autoload.cfg persists setting across power cycles (contains '1' or '0')
Program Auto-Start
  • Enabled: Loaded profile immediately begins running program steps
  • Disabled: Profile loads but waits for manual start command
  • Use Cases: OFF for development/testing, ON for unattended operation
Common Workflows
Scenario 1: Modify Existing Profile
  1. Click grey recent profile button → Loads into form
  2. Edit settings as needed
  3. Click "Upload to Device" → Sends modified profile to hardware
  4. Device runs the updated profile
Scenario 2: Run Specific Old Profile (Temporary)
  1. Click "Load from Device" → Shows available profiles
  2. Select desired profile → Device immediately loads and runs it
  3. Temporary only - profile runs until power cycle
  4. After restart → Device reverts to newest timestamped profile
Scenario 3: Make Old Profile Persistent
  1. Click "Load from Device" → Select old profile → Loads into form
  2. Click "Upload to Device" → Creates new timestamped file
  3. New timestamp makes it the "newest" for auto-load
  4. Future restarts will auto-load this profile
Scenario 4: Quick Profile Switch (Session Only)
  1. Use "Load from Device" to switch between stored profiles
  2. Each selection immediately changes device behavior
  3. No upload needed - profiles already exist on device
  4. Reset warning - power cycle reverts to newest timestamped profile
Troubleshooting
  • Form shows profile but device not responding: Profile only loaded in form - use "Upload to Device"
  • Wrong profile runs after restart: Auto-load uses newest timestamped profile, not last selected. Upload desired profile to make it newest.
  • Old profile stops running after power cycle: "Load from Device" is temporary. Upload the profile to make it persistent.
  • Profile loads but doesn't start: Program Auto-Start may be disabled - check device settings
  • Green highlighting disappears: Normal when device status updates - highlighting shows form state vs device state
  • Auto-load keeps loading wrong profile: Use "set autoload off" to disable (creates autoload.cfg), or upload correct profile with newer timestamp

インターフェース操作

  • デバイスラベル: ページ上部中央のバッジ（「Device 1」など）をクリックして、デバイス名をカスタマイズします。

    異なるブラウザタブで複数のデバイスを使用する場合、各タブに固有の識別子を付けることができます。バッジをクリックして「Device 1」、「Device 2」、「Lab A」などのカスタム名を設定します。

    デバイスラベルはページバッジとブラウザタブのタイトルの両方に表示され、複数のデバイスを簡単に区別できます。ラベルはタブごとに保存されます（グローバルではない）ので、各ブラウザタブが独自のデバイスIDを持つことができます。

    使用例：複数のタブを開き、それぞれ異なる物理デバイスに接続し、混乱を避けるためにそれに応じてラベルを付けます。

  • 背景色: 色付きスウォッチ（右上）をクリックして、ページの背景色を変更します。

    複数のデバイスタブを視覚的に区別するのに便利です—各デバイスに異なる色（緑、青、または赤）を割り当てると、どのタブで作業しているかをすばやく識別できます。色の設定はブラウザタブごとに保存されます。

  • シンプル/詳細モード: 簡素化されたビューと完全なインターフェースビューを切り替えます。

    シンプルモード：使用頻度の低いセクションを非表示にして、インターフェースをすっきりさせます。日常的な操作や、すでにシステムに精通している場合に最適です。

    詳細モード：利用可能なすべてのセクションと設定を表示します。完全な制御が必要な場合や、システムを学習している場合に使用します。

    歯車アイコン（⚙）：クリックして、シンプルモードで表示するセクションをカスタマイズします。設定は自動的に保存されます。

    ヒント：すべての機能を探索するには詳細モードから始め、定期的に使用するセクションがわかったらシンプルモードに切り替えます。

モニター / 操作

  • ステータスタイル: 任意のタイルをクリックしてプリセットまたはカスタム値を設定します。

    例：SimTime（日の出/正午またはHHMM）、速度（-300x～300xまたはホールド）、強度（0.0～1.0）。Program/Rotation/Dual Sunを切り替えるにはダブルクリックします。

  • 太陽アーク: 現在の時刻と昼夜のシェーディングを示す視覚的な太陽の経路。太陽の位置、色、強度もモデル化します。
  • コマンドコンソール: オートコンプリートサポート付きのコマンドを入力します。

    Tabキーを押してオートコンプリート候補を順に表示します。上下矢印キーを使用して過去10個のコマンドをナビゲートします。Enterキーを押してコマンドを送信します。コマンドは大文字と小文字を区別し、デバイスの動作を直接制御します。

  • すべてのコマンドを表示: 利用可能なすべてのコマンドの完全なリストをログウィンドウに表示します（詳細モードで表示）。
  • デバイス接続: Web Serial APIがUSB経由でデバイスに直接接続します。

    互換性のあるブラウザ（Chrome、Edge、Opera）が必要です。「接続」ボタンをクリックして、リストからデバイスを選択します。接続に失敗した場合は、適切なUSBドライバがインストールされていることを確認してください。アクティブな場合はステータスが「接続済み」と表示され、それ以外の場合は「未接続」と表示されます。

Solar Simulator User Guide

Welcome to the SolaSim Studio user guide. This document provides everything you need to set up your solar simulation hardware and use the web interface to control lighting, rotation, and imaging for your experiments.

🚀 Quick Start: Installation & Setup

1. Prepare the Device (RP2040:bit)

The Solar Simulator runs on a Raspberry Pi Pico-based board called the RP2040:bit. You need to install the custom firmware (a single .uf2 file) which contains both the operating system (MicroPython) and the Solar Simulator software.

  1. Download the Firmware: Obtain the latest SolarSimulator_YYYYMMDD.uf2 file from the SolaSim GitHub repository.
  2. Enter Bootloader Mode:
    • Hold down the BOOT button on the edge of the board.
    • While holding BOOT, press the RESET button on the edge of the board.
    • Release the button.
  3. Install Firmware:
    • A new drive named RPI-RP2 will appear on your computer (like a USB stick).
    • Drag and drop the .uf2 file onto this drive.
    • The device will automatically disconnect and reboot. The installation is complete!

2. Open SolaSim Studio

You do not need to install any software on your computer to control the device.

  • Recommended: Open https://odeepllama.github.io/SolarSim/ in a compatible browser (Google Chrome, Microsoft Edge, or Opera).
  • Advanced/Offline: If you have the ProfileBuilder.html file locally, you can simply double-click it to open it in your browser.

3. Connect

  1. After connecting the RP2040 board to your computer, click the green Connect button in the top "Monitor / Interact" bar.
  2. A browser window will pop up asking permission to connect to a serial device.
  3. Select your device (often listed as "Board in FS node") and click Connect.
  4. The status badge will turn green and show CONNECTED.
  5. If the device does not immediately connect, press the RESET button on the edge of the board and try connecting again.

💡 Core Concepts

Before using the system, it is helpful to understand two key concepts:

Form vs. Device

  • The Form (Web Page): This is your workspace. Changing numbers or settings here does not immediately change the device. It is like writing a document before printing it.
  • The Device (Hardware): This is the physical board running the simulation.
  • Syncing: To make your form settings take effect on the hardware, you must click Upload to Device.

Auto-Load & Persistence

  • Temporary Changes: If you use the "Monitor" tiles to change speed or intensity, those changes are lost if the device loses power.
  • Persistent Profiles: When you Upload to Device, the system saves your settings as a file.
  • Auto-Load: By default, when the device powers up, it looks for the most recently uploaded profile and runs it automatically. If Auto-Load is disabled, the device will start with default settings (single sun, 1X speed, sunrise 0600, sunset 1800, etc).

🎮 Three Ways to Use SolaSim Studio

1. Real-Time Interaction (Monitor Mode)

Best for: Quick adjustments, testing hardware, and checking status.

At the top of the page is the Monitor / Interact bar. Click the arrow to expand it if the Solar Arc is not visible.

  • Status Tiles: You will see live data from the device (e.g., SimTime, Speed, Intensity).
    • Click a tile to change its value immediately. For example, click Speed to pause (HOLD) or fast-forward (600x) the simulated time.
    • Double-click certain tiles (like Program or Rotation) to toggle them ON or OFF.
  • Command Console: Type commands manually (e.g., set intensity 0.5) in the text box for precise control. The Show All Commands button will output all commands in the log window (Visible in Advanced mode).
  • Solar Arc: A visual representation of the sun's position and day/night cycle.
Note: Changes made here happen immediately but are not saved to the permanent profile.

2. Building Custom Profiles

Best for: Designing specific experiments with precise lighting and timing requirements.

Use the main form to define your experiment parameters.

Core Settings

  • Start Time: When the simulation clock begins (e.g., 0600 for 6 AM).
  • Time Scale: How fast time passes (e.g., 60x means 1 hour passes in 1 minute).
  • Solar Mode:
    • BASIC: Simple sunrise (6 AM) to sunset (6 PM) with constant sun size.
    • SCIENTIFIC: Calculates accurate sun position based on the Date and Latitude you provide.
  • Sun Color: Choose BLUE (standard for experiments), NATURAL (yellow/white), or CUSTOM (RGB).

Program Steps (The Timeline)

If you need the conditions to change over time (e.g., "Run at 2x speed until noon, then hold for 1 hour"), use the Program Steps section.

  1. Ensure Program Enabled is checked.
  2. Click Add Step to create a new sequence.
  3. For each step, define:
    • Time: When this step targets (e.g., 1200).
    • Transition: RUN (simulate time passing) or JUMP (skip instantly).
    • Speed/Intensity: The conditions during this step.
  4. Use the Timeline Visualization to verify your sequence looks correct.
  5. You can add additional days with different steps/conditions if your experiment requires.

Rotation & Imaging

Configure the turntable and camera triggers.

  • Rotation Enabled: Master switch for the rotating platform.
  • Capture Mode: STILLS (stop-motion) or VIDEO (continuous).
  • Intervals: How often to take photos or rotate.

3. Uploading & Running

Best for: Starting your experiment.

Once your form is configured:

  1. Check the Profile Diff Viewer (yellow highlights). This shows you what is different between your form and the device.
  2. Click the green Upload to Device button.
  3. Follow the prompt (you may need to press the RESET button on the device if requested, though usually it is automatic).
  4. The device will save the profile, restart, and begin running your experiment immediately.

📂 Managing Profiles

Loading from Device

If you want to see or edit the profile currently running on the hardware:

  1. Click Load / Delete from Device.
  2. A list of files stored on the RP2040 will appear.
  3. Select a profile to load it into the web form.
    Note: Selecting a profile here also tells the device to run it immediately, but it does not make it the "default" for the next startup unless you re-upload it or it is already the most recent profile.

Saving & Importing

  • Download Profile: Saves your current settings as a text file (.txt) to your computer. Useful for sharing experiments or backups.
  • Import Profile: Loads a .txt file from your computer into the web form.
  • Recent Profiles: The bar at the top of the form saves your recent configurations automatically. Click one to restore it. Click the Star to prevent key profiles from rotating off the screen as more are created.

Profile Comparison

When you load a profile, the Comparison Window will show you exactly what is different between your current settings and the new profile.

  • Orange Fields: Indicate settings that differ.
  • Apply: Overwrites your form with the new profile.

⚙️ Advanced Features

Simple vs. Advanced Mode

  • Simple Mode: Hides complex settings (like specific servo timings or RGB values) to keep the interface clean.
  • Advanced Mode: Shows all available parameters.
  • Use the Gear Icon (⚙) next to the mode toggle to customize which sections are hidden in Simple Mode.

Device Labeling

If you have multiple devices connected:

  1. Click the Device Badge (e.g., "Device 1") in the top header.
  2. Rename it (e.g., "Experiment A").
  3. Use the Background Color swatches (top right) to color-code your tabs (Green, Blue, Red).

Troubleshooting

Issue Solution
Device not connecting Ensure no other tab or program (like Thonny or VS Code) is using the USB port. Unplug and replug the device.
Settings didn't change Did you click Upload? Changing the form only updates the web view, not the hardware.
Wrong profile on startup The device auto-loads the profile with the newest timestamp. If you loaded an older profile via "Load from Device", it won't persist after a reboot. To make an old profile permanent, Upload it again so it gets a new timestamp.

ソーラーシミュレーター ユーザーガイド

SolaSim Studio ユーザーガイドへようこそ。このドキュメントは、ソーラーシミュレーションハードウェアのセットアップ方法と、ウェブインターフェースを使った照明、回転、イメージング制御の手順をすべて説明します。

🚀 クイックスタート：インストールとセットアップ

  1. デバイスの準備（RP2040:bit）
    ソーラーシミュレーターはRP2040:bitというRaspberry Pi Picoベースのボード上で動作します。カスタムファームウェア（.uf2ファイル）をインストールする必要があります。
    1. ファームウェアのダウンロード： SolaSim GitHubリポジトリから最新のSolarSimulator_YYYYMMDD.uf2ファイルを取得します。
    2. ブートローダーモードに入る：
      • 基板の端にあるBOOTボタンを押し続けます。
      • BOOTを押したまま、RESETボタンを押します。
      • ボタンを離します。
    3. ファームウェアのインストール：
      • コンピュータに「RPI-RP2」という新しいドライブが表示されます（USBメモリのように）。
      • このドライブに.uf2ファイルをドラッグ＆ドロップします。
      • デバイスは自動的に切断・再起動します。これでインストール完了です！
  2. SolaSim Studioを開く
    デバイスを操作するためにPCへソフトウェアをインストールする必要はありません。
    • 推奨：https://odeepllama.github.io/SolarSim/ をGoogle Chrome、Microsoft Edge、Operaなどの対応ブラウザで開きます。
    • 上級者/オフライン：ProfileBuilder.htmlファイルをローカルでダブルクリックしてブラウザで開くことも可能です。
  3. 接続
    RP2040ボードをPCに接続後、画面上部の「Monitor / Interact」バーの緑色の「接続」ボタンをクリックします。
    1. ブラウザのウィンドウが開き、シリアルデバイスへの接続許可を求められます。
    2. デバイス（多くの場合「Board in FS node」等）を選択し、「接続」をクリックします。
    3. ステータスバッジが緑色になり「CONNECTED」と表示されます。
    4. すぐに接続できない場合は、基板のRESETボタンを押して再度お試しください。

💡 基本概念

  • フォームとデバイス
    フォーム（ウェブページ）: ここが作業スペースです。数値や設定を変更しても、すぐにデバイスへ反映されるわけではありません。印刷前の文書作成のようなものです。
    デバイス（ハードウェア）: 実際にシミュレーションを実行する物理ボードです。
    同期: フォームの設定をハードウェアに反映させるには「アップロード」ボタンをクリックしてください。
  • 自動ロードと永続性
    一時的な変更: 「Monitor」タイルで速度や強度を変更した場合、電源が切れるとその変更は失われます。
    永続プロファイル: 「アップロード」時、設定はファイルとして保存されます。
    自動ロード: 通常、デバイス起動時は最新のプロファイルが自動で読み込まれます。自動ロードを無効にすると、デフォルト設定（単一太陽、1X速度、日の出0600、日の入り1800等）で起動します。

🎮 SolaSim Studioの3つの使い方

  1. リアルタイム操作（モニターモード）
    最適：素早い調整、ハードウェアのテスト、状態確認
    ページ上部の「Monitor / Interact」バーを展開し、Solar Arcが見えない場合は矢印をクリックしてください。
    • ステータスタイル: デバイスからのライブデータ（SimTime、Speed、Intensity等）が表示されます。
    • タイルをクリックすると値を即座に変更できます（例：Speedで一時停止や高速化）。
    • 一部タイル（ProgramやRotation等）はダブルクリックでON/OFF切替が可能です。
    • コマンドコンソール: テキストボックスにコマンド（例：set intensity 0.5）を手入力できます。「Show All Commands」ボタンで全コマンドをログウィンドウに出力（Advancedモード時）。
    • Solar Arc: 太陽の位置や昼夜サイクルを視覚的に表示します。
    • ※ここでの変更は即時反映されますが、永続プロファイルには保存されません。
  2. カスタムプロファイルの作成
    最適：実験に合わせた詳細な照明・タイミング設計
    メインフォームで実験パラメータを定義します。
    • 開始時刻: シミュレーション開始時刻（例：0600は午前6時）
    • タイムスケール: 時間の進み方（例：60xは1分で1時間進行）
    • ソーラーモード:
      • BASIC：単純な日の出（6時）～日の入り（18時）、太陽サイズ一定
      • SCIENTIFIC：日付・緯度に基づき正確な太陽位置を計算
    • 太陽色: BLUE（実験標準）、NATURAL（黄/白）、CUSTOM（RGB指定）

    プログラムステップ（タイムライン）

    • 条件を時間で変化させたい場合（例：「正午まで2x速度、その後1時間HOLD」等）はProgram Stepsを利用します。
    • Program EnabledをONにします。
    • Add Stepで新しいシーケンスを追加します。
    • 各ステップで以下を設定：
      • Time：このステップの時刻（例：1200）
      • Transition：RUN（時間進行）またはJUMP（即時遷移）
      • Speed/Intensity：このステップ中の条件
    • Timeline Visualizationでシーケンスを確認できます。
    • 必要に応じて複数日・異なる条件も追加可能です。

    回転・イメージング

    • 回転有効: ターンテーブルの主電源
    • キャプチャモード: STILLS（静止画）またはVIDEO（連続）
    • インターバル: 撮影や回転の間隔
  3. アップロードと実行
    最適：実験開始時
    1. フォーム設定後、「Profile Diff Viewer」（黄色ハイライト）でフォームとデバイスの差分を確認します。
    2. 緑色の「アップロード」ボタンをクリックします。
    3. プロンプトに従い、必要に応じてRESETボタンを押してください（通常は自動）。
    4. デバイスがプロファイルを保存し、再起動して実験を開始します。

📂 プロファイル管理

デバイスからの読み込み

  1. 現在ハードウェアで動作中のプロファイルを確認・編集したい場合：
  2. 「Load / Delete from Device」をクリックします。
  3. RP2040上のファイル一覧が表示されます。
  4. プロファイルを選択するとウェブフォームに読み込まれます。
    ※ここで選択したプロファイルは即時実行されますが、再起動後もデフォルトにしたい場合は再度アップロードしてください。

保存とインポート

  • プロファイルのダウンロード: 現在の設定をテキストファイル（.txt）としてPCに保存。実験共有やバックアップに便利です。
  • プロファイルのインポート: .txtファイルをPCからウェブフォームに読み込みます。
  • 最近のプロファイル: フォーム上部のバーに最近の設定が自動保存されます。クリックで復元、スターで重要プロファイルを固定可能。

プロファイル比較

  • プロファイルを読み込むと、Comparison Windowで現在の設定と新しいプロファイルの違いが明確に表示されます。
  • オレンジ色のフィールド: 設定が異なる箇所を示します。
  • 適用: 新しいプロファイルでフォームを上書きします。

⚙️ 高度な機能

シンプルモードとアドバンストモード

  • シンプルモード: 複雑な設定（サーボタイミングやRGB値等）を非表示にし、インターフェースを簡潔に保ちます。
  • アドバンストモード: すべてのパラメータを表示します。
  • モード切替横のギアアイコン（⚙）でシンプルモード時に非表示にするセクションをカスタマイズ可能です。

デバイスラベリング

  • 複数デバイス接続時：
  • 上部ヘッダーのDevice Badge（例：「Device 1」）をクリック
  • 名前を変更（例：「Experiment A」）
  • 右上の背景色スウォッチ（緑・青・赤）でタブを色分け
